# GCP Infrastructure & Cost Optimization

To support our growth while maintaining fiscal responsibility, we must optimize our Google Cloud Platform (GCP) resources. This section details our infrastructure audit, cost leaks, and the strategy to transition to a high-performance, cost-effective OpenLiteSpeed setup.

## The Best-Practice Architecture

Instead of managing multiple idle VMs through third-party control panels like RunCloud, we are consolidating our entire stack onto a single, high-performance, custom-configured VM instance running **OpenLiteSpeed**.

### Why OpenLiteSpeed?
1. **Event-Driven Architecture:** OpenLiteSpeed is extremely fast, handling tens of thousands of concurrent connections with minimal CPU and memory usage.
2. **Built-in LSCache:** The LiteSpeed Cache plugin for WordPress connects directly to the server's cache module, delivering sub-100ms page loads without heavy Redis/Nginx overhead.
3. **HTTP/3 & QUIC Support:** Out-of-the-box support for the latest web protocols, improving mobile loading speeds significantly.
4. **Multi-Tenant Friendly:** Easily hosts production, staging, and ephemeral "Remote Dev" preview environments on a single instance.

### Target VM Specification
* **Instance Type:** `e2-standard-4` (4 vCPUs, 16 GB Memory)
* **Operating System:** Ubuntu 24.04 LTS
* **Web Server:** OpenLiteSpeed
* **Database:** MariaDB 10.11 (optimized for WordPress)
* **Estimated Monthly Cost:** ~$120.00 / month
