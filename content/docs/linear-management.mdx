## Git & Linear Integration

We link our codebase changes directly to Linear issues using a strict branch naming convention.

### Branch Naming Convention
Every Git branch must be prefixed with the Linear Issue ID (e.g., `MYR-XX`) followed by a short, kebab-case description:

* **Features:** `feature/MYR-54-experience-upload-bug`
* **Bug Fixes:** `bugfix/MYR-55-price-display-free`
* **Chore/Infrastructure:** `chore/MYR-60-revert-wp-version`

### Automated State Transitions
When you create a Pull Request on GitHub:
1. Include the Linear Issue ID in the PR title (e.g., `[MYR-54] Fix experience listing image upload`).
2. Include the Linear link in the description.
3. Our integration will automatically transition the Linear issue to **In Review**.
4. When the PR is merged into `staging`, the issue is moved to **Staging/QA**.
5. Once verified on staging and released to production, the issue is moved to **Done**.
